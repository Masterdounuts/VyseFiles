### Problem
- Control UI session dropdown showed 64+ sessions (clutter)

- Root cause: Cron jobs create `:run:` variants, old sessions don't auto-expire

### What We Did

| Action | Result |

| Pruned Quartermaster | 28 → 1 (Stock Price Monitor cron) |

### Systems Created

1. **Session Cleanup Cron** - Daily 4am UTC
   - Removes ALL `:run:` entries (cron run variants)
   - Removes orphaned `:main` (no label + no lastChannel)
   - Job: `Shipwright: Session Cleanup`

2. **Session Health Monitor Cron** - Hourly
   - Checks context usage (totalTokens / contextTokens)
   - Alerts if >80% used
   - Job: `Shipwright: Session Health Monitor`
