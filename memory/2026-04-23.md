## Browser Tool Expertise (Apr 23, 2026 00:33 UTC)

**Source:** https://docs.openclaw.ai/tools/browser

**What it is:**
- Dedicated Chrome/Brave/Edge/Chromium profile controlled by the agent
- Isolated from personal browser, managed through Gateway
- Two profiles: `openclaw` (managed/isolated) vs `user` (real Chrome via MCP)

**CLI Commands:**
```bash
openclaw browser --browser-profile openclaw status
openclaw browser --browser-profile openclaw start
openclaw browser --browser-profile openclaw open https://example.com
openclaw browser --browser-profile openclaw snapshot
```

**Key Config (openclaw.json):**
```json
{
  "browser": {
    "enabled": true,
    "defaultProfile": "openclaw",
    "profiles": {
      "openclaw": { "cdpPort": 18800, "color": "#FF4500" },
      "user": { "driver": "existing-session", "attachOnly": true },
      "work": { "cdpPort": 18801, "color": "#0066CC" }
    }
  }
}
```

**Common Issue - Browser Missing:**
- If `openclaw browser` is unknown → check `plugins.allow` includes "browser"
- Both `plugins.entries.browser.enabled` AND `plugins.allow` need browser

**Profile Types:**
- `openclaw`: Managed, isolated browser (default)
- `user`: Real Chrome via Chrome DevTools MCP (reuse login state)
- `remote`: Explicit CDP URL to browser running elsewhere
- `browserless`: Hosted service (Browserless, Browserbase)

**Remote CDP Providers:**
- Browserless: `wss://production-sfo.browserless.io?token=<API_KEY>`
- Browserbase: `wss://connect.browserbase.com?apiKey=<API_KEY>`

**Security:**
- Loopback-only by default
- Uses shared-secret auth (gateway token or password)
- Keep Gateway on private network (Tailscale)
- Treat remote CDP tokens as secrets
